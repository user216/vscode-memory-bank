# ADR-0019: Tester Feedback ‚Äî New MCP Tools, Sidebar Notes, v1 Migration

## Context

Real-world testing of Memory Bank v2 by a tester who migrated 89 tasks + 24 decisions uncovered several gaps:

1. **No bulk status update** ‚Äî updating many items one-by-one was painful
2. **Tags were unused infrastructure** ‚Äî `memory_tags` could list them but no tool could add them
3. **Status parsing fragile** ‚Äî YAML frontmatter `status:` field, `**Status:**` bold, and `## Status:` heading formats not handled consistently across MCP tools and extension sidebar
4. **Status vocabulary mismatches** ‚Äî testers used "Done", "Open", "Draft" instead of canonical values
5. **Orphaned v1 artifacts** ‚Äî migration left `tasks/` and `decisions/` subdirectories, stale NOTE files, and SQLite DB
6. **No Notes section in sidebar** ‚Äî extension showed Files, Tasks, Decisions but not Notes
7. **`memory_recall` parameter discovery** ‚Äî description didn't list valid parameters, causing trial-and-error
8. **`memoryBank.init` still scaffolded v1 layout** ‚Äî created `tasks/`, `decisions/` subdirectories

## Decision

Address all tester feedback in a single batch:

### New MCP tools (3)
- **`memory_bulk_update_status`** ‚Äî accepts array of `{id, status, log_entry?}`, max 50 items, reports partial success/failure
- **`memory_add_tag`** ‚Äî adds tags to YAML frontmatter, handles inline `[tag1, tag2]` and YAML list `- tag` formats, creates frontmatter if none exists
- **`memory_migrate_v1`** ‚Äî scans `tasks/` and `decisions/` subdirs, moves files to root, adds YAML frontmatter, `dry_run` mode by default

### Status parsing standardization
- **3-tier pattern** applied consistently: YAML frontmatter `status:` ‚Üí `**Status:**` bold ‚Üí `## Status:` heading ‚Üí fallback
- **`STATUS_ALIASES`** in `shared-utils.ts`: `Done‚ÜíCompleted`, `Open‚ÜíPending`, `Draft‚ÜíProposed`, `Approved‚ÜíAccepted`
- Extension sidebar providers (`tasks-provider.ts`, `decisions-provider.ts`) updated with same 3-tier pattern

### Extension sidebar
- **`NotesTreeProvider`** ‚Äî new sidebar section showing `NOTE-*.md` files with tags as description
- **`memoryBank.init`** updated ‚Äî creates v2 flat layout with YAML frontmatter, no subdirectories

### Cleanup
- Deleted orphaned `NOTE-001.md`, `NOTE-002.md`, `NOTE-003.md` (duplicates of core files)
- Deleted stale `.mcp/memory-bank.db` (SQLite from v1)
- Simplified `.gitignore` (removed SQLite rules)
- Updated `memory_recall` description to list valid parameters explicitly
- Updated `memory_schema` to include all 20 tools

## Consequences
- MCP server now has 20 tools (was 17)
- Testers can batch-update status and add tags via MCP tools
- v1‚Üív2 migration has a dedicated tool with dry-run safety
- Extension sidebar shows all 4 item types (Files, Tasks, Decisions, Notes)
- Status parsing is consistent across all three layers (MCP parser, MCP update tools, extension sidebar)
