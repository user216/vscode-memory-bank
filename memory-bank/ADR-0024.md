# ADR-0024: Tag filter for Tasks sidebar + file ID as label

## Context
Tasks in the sidebar had no way to filter by tag — users with many tasks and different tags had to scroll through the full list. Additionally, the previous ADR-0023 set the frontmatter title as the tree item label, but users preferred seeing the file ID (e.g., `TASK-001`) first for quick identification, with the title in the description.

## Decision
1. **Tag filter**: Add a filter button (`$(filter)` icon) to the Tasks view title bar. Clicking it opens a VS Code QuickPick listing all unique tags found across task files. Selecting a tag filters the tree to only show matching tasks. A clear button (`$(close)` icon) appears when a filter is active. The view description shows `#tagname` during filtering.

2. **File ID as label**: Swap label and description in all three sidebar providers (Tasks, Decisions, Notes) — show the file ID as the label and the title/status/tags as the description.

3. **Implementation details**:
   - `TasksTreeProvider` gains `activeTagFilter` state, `filterByTag()`, `clearTagFilter()`, and `getAllTags()` methods
   - Tasks view uses `createTreeView` instead of `registerTreeDataProvider` to access the `TreeView.description` property
   - New context key `memoryBank.taskTagFilterActive` controls clear button visibility
   - Two new commands: `memoryBank.filterTasksByTag`, `memoryBank.clearTaskTagFilter`

## Alternatives Considered

### Right-click context menu on tags
VS Code tree views don't support dynamic submenus, so listing tags per-item in a context menu would require a separate command per tag or a generic "filter by this task's tag" that doesn't let users pick from all tags.

### Inline tag chips as separate tree items
Render each tag as a child tree item users could click. Rejected because it clutters the tree with non-task items and conflicts with the existing relation children.

### Filter all views (Tasks + Decisions + Notes)
Could add tag filtering to all three views. Deferred — Tasks is the primary use case. Can extend to Decisions and Notes in a follow-up if needed.

## Consequences
- Users can quickly narrow down tasks by tag with two clicks (filter icon → tag name)
- File IDs are visible at a glance for all item types, making items easier to identify and reference
- The `createTreeView` handle opens the door for future features (badges, reveal, etc.)
- Filter state is in-memory only — resets on extension reload (intentional, not persistent)
