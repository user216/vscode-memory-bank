# ADR-0013: Distribute as Copilot Agent Plugin via Git submodule

**Status:** Proposed
**Date:** 2026-03-27
**Deciders:** 

## Context
VS Code Agent Plugins (Preview, March 2026) are a new distribution model — Git repos with plugin.json, skills, agents, hooks, and MCP configs. The memory-bank toolkit already has all the assets but distributes them only as a monorepo or VSIX.

## Decision
Create a separate Git repo (user216/memory-bank-plugin) containing all plugin assets and add it as a submodule at copilot-plugin/. The plugin bundles skills, agents, hooks, prompts, and instructions only — the MCP server is delivered exclusively by the VS Code extension (ADR-0017). This gives dual distribution: VSIX extension for the full VS Code UI experience + MCP server, and Agent Plugin for zero-config Copilot agent integration with skills and agents.

## Alternatives Considered

_None documented._

## Consequences
- Plugin repo (memory-bank-plugin) must be kept in sync when skills/agents/hooks/prompts change in the monorepo
- Users need the VS Code extension installed separately for MCP tools
- Plugin is discoverable via @agentPlugins in VS Code once added to a marketplace repo
- Dual distribution: VSIX for UI features + MCP server, Agent Plugin for skills/agents/hooks
- Plugin is pure content (no native deps, no setup scripts) — works on all platforms without compilation
