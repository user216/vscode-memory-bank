# ADR-0020: Eager placeholder view registration to prevent startup errors

## Context

VS Code restores sidebar views from the previous session during startup, before the extension's async `initializeFullUI()` completes. When the user had Notes/Tasks/Decisions views open, VS Code tries to render them but the tree data providers aren't registered yet, causing "No view is registered with id: memoryBankNotes" error notifications.

## Decision

Register placeholder (empty) tree data providers immediately in the synchronous `activate()` function for all 5 view IDs (`memoryBankFiles`, `memoryBankTasks`, `memoryBankDecisions`, `memoryBankNotes`, `memoryBankMcpSetup`). The placeholders return empty children, satisfying VS Code's view restoration. Once `initializeFullUI()` completes, the placeholders are disposed and replaced by real providers with actual data.

## Consequences

- Eliminates "No view is registered" errors on startup
- Sidebar briefly shows empty views before real data loads (sub-second, imperceptible)
- No behavior change once fully initialized
