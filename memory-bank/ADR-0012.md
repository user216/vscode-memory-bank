# ADR-0012: Automate extension setup — zero manual steps after install

## Context
A macOS user installed the VSIX and got nothing — no sidebar icon, no status bar, no MCP config. The extension required manual CLI steps to build the MCP server and manual .mcp.json creation.

## Decision
Switch to `onStartupFinished` activation with a two-mode pattern: show a welcome view with [Initialize Memory Bank] button on uninitialized workspaces, full UI on initialized ones. Bundle the compiled MCP server inside the VSIX. Auto-generate `.vscode/mcp.json` with correct absolute paths from `context.extensionPath`.

## Alternatives Considered

### Keep manual setup with better docs
- **Rejected:** Users expect extensions to just work after install.

### Bundle node_modules in VSIX (pre-built native binaries)
- **Rejected:** Platform-specific binary. Solved by [[ADR-0016]] (zero native deps).

### Publish MCP server as separate npm package
- **Rejected:** Adds another install step.

## Consequences
- Extension activates on every workspace (`onStartupFinished`)
- VSIX includes MCP server JS + pure JS deps (no native compilation)
- MCP config paths auto-update when extension version changes
