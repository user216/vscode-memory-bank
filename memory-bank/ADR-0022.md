# ADR-0022: Rename MCP server key from "memory-bank" to "mbank-{workspace}"

## Context
The MCP server key `"memory-bank"` in `.vscode/mcp.json` conflicts with Claude's reserved `#memory` prefix. When users type `#memory-bank` in chat to reference the MCP server, Claude intercepts it as a memory-related command. Additionally, all workspaces using the extension had the same generic server key, which could cause confusion in multi-workspace setups.

## Decision
Rename the MCP server key to `mbank-{workspace}` where `{workspace}` is the sanitized workspace folder name (lowercase, alphanumeric + hyphens). For example, workspace `my-project` produces key `mbank-my-project`. The config generator auto-migrates existing `"memory-bank"` keys to the new format, preserving user `env` settings. The `deriveServerKey()` function handles sanitization. For copy-to-clipboard snippets without workspace context, the fallback key is `mbank`.

## Alternatives Considered

### Keep memory-bank key
Would continue causing #memory prefix conflicts in Claude chat. No resolution for the reported issue.

### Use a random hash suffix (mbank-abc123)
Avoids conflicts but produces opaque names that are hard to identify across workspaces or in documentation.

### Use a timestamp suffix (mbank-1711612800)
Unique but unreadable. Users can't tell which workspace a key belongs to.

## Consequences
- Existing .vscode/mcp.json files with 'memory-bank' key are automatically migrated on next extension activation
- Users can now type #mbank-{workspace} in Claude chat without triggering reserved prefix behavior
- Each workspace gets a unique server key, preventing cross-workspace confusion
- Documentation updated to use 'mbank' as the server key in examples
- Copy-to-clipboard snippet uses mbank-{workspace} when workspace is known, mbank otherwise
