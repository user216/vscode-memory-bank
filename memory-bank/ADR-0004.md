# ADR-0004: No Model Version Pinning in Agent Files

## Context
Agent `.agent.md` files accept a `model` field that pins a specific AI model version. VS Code does not support variable substitution in `.agent.md` frontmatter — the model field is static text. We previously hardcoded versions and had to update all agent files each time a new model released.

## Decision
Remove the `model` field from all agent files. Agents inherit whatever model the user has configured as their default. A `memory-bank-config.json` documents the recommended model. An optional `scripts/update-model.sh` exists for explicit pinning.

## Alternatives Considered

### Hardcode latest version everywhere
- **Rejected because:** Already failed twice (Opus 4 → Opus 4.5 → Opus 4.6). Will fail again.

### Centralized config + mandatory propagation script
- **Rejected because:** Adds a manual step that's easy to forget.

### Remove model field + no config file
- **Rejected because:** The config file costs nothing and serves as documentation.

## Consequences
- Agents always use the user's current default model — zero maintenance on upgrades
- `memory-bank-config.json` documents the recommended model for reference
- `scripts/update-model.sh` is available for users who want explicit pinning
