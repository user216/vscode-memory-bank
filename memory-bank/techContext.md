# Tech Context

## Target Platform
- **IDE**: VS Code (1.106+)
- **AI Extension**: GitHub Copilot extension with Claude Agent SDK (agent mode)
- **AI Models**: Claude Opus (latest)
- **OS**: Linux (primary), macOS, Windows

## Tech Stack

### MCP Server (v2.0.0)
- **Runtime**: Node.js 20+
- **Language**: TypeScript 5.x, compiled with `tsc`
- **Index**: In-memory `Map<string, ParsedItem>` + MiniSearch (7KB, pure JS)
- **Frontmatter**: `gray-matter` (30KB, pure JS) — parses YAML `---` delimited metadata
- **Full-text search**: MiniSearch with BM25 ranking, prefix/fuzzy matching, title boosting
- **Graph**: Adjacency lists (`outgoing`/`incoming` Maps) for relationship traversal
- **File watching**: `chokidar` for incremental index updates
- **Protocol**: `@modelcontextprotocol/sdk` (stdio transport)
- **Zero native dependencies** — no `better-sqlite3`, no `node-gyp`, no C/C++ compilation
- **17 tools**: search, query, recall, link, unlink, update-link, graph, schema, create-task, create-decision, create-note, update-status, update-decision, import-decisions, save-context, status, tags

### VS Code Extension (v0.3.1)
- **Language**: TypeScript, bundled with `esbuild`
- **API**: VS Code Extension API
- **MCP server bundled**: Pure JS deps included at build time (no runtime `npm install`)
- **Config generator**: Always refreshes MCP server path on activation, preserves user env
- **Dual-layout support**: v1 subdirectories + v2 flat layout
- **YAML frontmatter parsing**: Regex-based (no `gray-matter` in extension bundle)

### Copilot Plugin (v0.4.0)
- Skills + agents + hooks + prompts only (no bundled MCP — [[ADR-0017]])
- Distributed via git submodule

## MCP Configuration
The VS Code extension auto-generates `.vscode/mcp.json` for GitHub Copilot on activation.
For Claude Code or other tools, use the MCP Setup sidebar to copy the config snippet.

## VS Code Customization Primitives
| Primitive | Location | Format |
|-----------|----------|--------|
| Custom Instructions | `instructions/` | `.instructions.md` |
| Agent Skills | `skills/` | `SKILL.md` + resources |
| Prompt Files | `.github/prompts/` | `.prompt.md` |
| Custom Agents | `.github/agents/` | `.agent.md` |
| Hooks | `.github/hooks/` | `.json` |
| MCP Servers | `.vscode/mcp.json` | JSON (auto-generated by extension) |

## Dependencies
- Original instruction: [github/awesome-copilot/instructions/memory-bank.instructions.md](https://github.com/github/awesome-copilot/blob/main/instructions/memory-bank.instructions.md)
- CodeGraphContext: external end-user MCP tool ([[ADR-0011]])

## Constraints
- All memory bank files must be valid markdown with YAML frontmatter
- Must maintain backward compatibility with v1 `**Status:** X` metadata format
- MCP server must have zero native dependencies ([[ADR-0016]])
- Extension must bundle MCP server deps in VSIX ([[ADR-0018]])
